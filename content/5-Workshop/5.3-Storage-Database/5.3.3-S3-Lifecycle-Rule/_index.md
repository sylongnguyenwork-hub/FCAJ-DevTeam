#### Automate Raw Document Deletion (Cost & Security Optimization)
Once a document is processed by Textract AI and the extracted data is safely stored in DynamoDB, keeping the original image/PDF files on S3 is unnecessary and incurs ongoing storage costs. We will configure an **S3 Lifecycle Rule** to automatically delete these files after 1 day.

1. Navigate to the **Amazon S3** console and click on your ingestion bucket: `idp-ingest-bucket-1` (or `idp-ingest-bucket-01`).
2. Switch to the **Management** tab and click the **Create lifecycle rule** button.
3. **Lifecycle rule name**: Enter `Auto-Delete-Raw-Invoices`.
4. **Choose a rule scope**: Select **Limit the scope of this rule using one or more filters**.
5. **Prefix**: Enter `uploads/` (This is the folder path generated by the React Frontend during upload).

![Lifecycle Rule](/images/5-Workshop/5.3-Storage-Database/lifecycle-rule-1.png)

6. Scroll down to the **Lifecycle rule actions** section and check the box for **Expire current versions of objects**.
7. In the newly expanded **Expire current versions of objects** section, under **Days after object creation**: Enter `1`.

![Lifecycle Rule](/images/5-Workshop/5.3-Storage-Database/lifecycle-rule-2.png)

8. Click **Create rule** to save the configuration.
